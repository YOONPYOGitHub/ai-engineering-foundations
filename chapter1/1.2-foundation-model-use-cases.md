## Use Case 개요

### 8가지 주요 카테고리

Foundation Models는 범용적이기 때문에 다양한 use case에 적용 가능합니다.

**분류:**
```
개발자 도구:
├─ Coding

창작 도구:
├─ Image/Video Production
└─ Writing

학습 도구:
└─ Education

커뮤니케이션:
└─ Conversational Bots

정보 처리:
├─ Information Aggregation
└─ Data Organization

비즈니스:
└─ Workflow Automation
```

### 조사 방법

**Enterprise:**
- 50개 기업 인터뷰
- 100+ 케이스 스터디

**Consumer:**
- GitHub 205개 오픈소스 앱 분석 (500+ stars)

**분포 (오픈소스 앱 기준):**
```
Coding:              가장 많음
Conversational:      두 번째
Image/Video:         세 번째
Writing:             적음 (기업용이 많아서)
Education:           적음 (기업용이 많아서)
Data Organization:   적음 (기업용이 많아서)
```

### 기업 채택 패턴

**a16z Growth Report (2024):**
```
내부용 애플리케이션 > 외부용 애플리케이션

이유:
- 리스크 최소화
- 데이터 프라이버시
- 컴플라이언스
- 실패 영향 제한

예:
내부 지식 관리 > 고객 지원 챗봇
```

**Close-ended > Open-ended:**
```
분류 태스크 선호:
- 평가하기 쉬움
- 리스크 추정 용이
- 실패 범위 명확

Foundation Models는 open-ended 가능하지만
실전에서는 close-ended로 많이 사용
```

## 1. Coding

> **"AI 설문조사에서 압도적 1위 use case"**

### 왜 인기있는가?

1. **AI가 코딩을 잘함**
2. **초기 AI 엔지니어 = 개발자**
   - 코딩 문제에 노출
   - 직접 경험

### 성공 사례

**GitHub Copilot:**
- 최초의 프로덕션 성공 사례 중 하나
- 출시 2년 만에 $100M ARR
- Code completion 도구

**스타트업 투자 (2024년 8월):**
- Magic: $320M 투자
- Anysphere: $60M 투자

**오픈소스:**
- gpt-engineer: 1년 내 50K stars
- screenshot-to-code: 1년 내 50K stars

### 주요 코딩 태스크

**1. 웹/PDF에서 구조화된 데이터 추출**
- AgentGPT

**2. English → Code**
- DB-GPT: 자연어 → SQL
- SQL Chat: 대화형 SQL
- PandasAI: 자연어 → Pandas

**3. 디자인/스크린샷 → 코드**
- screenshot-to-code
- draw-a-ui
- 디자인 보고 웹사이트 코드 생성

**4. 언어/프레임워크 번역**
- GPT-Migrate
- AI Code Translator

**5. 문서화**
- Autodoc

**6. 테스트 생성**
- PentestGPT

**7. Commit 메시지**
- AI Commits

### AI는 엔지니어를 대체할까?

**낙관론 (대체):**

Jensen Huang (NVIDIA CEO):
> "AI가 소프트웨어 엔지니어를 대체할 것  
> 아이들에게 코딩을 가르치라고 하지 말라"

Matt Garman (AWS CEO):
> "가까운 미래에 대부분의 개발자가 코딩을 멈출 것  
> (소프트웨어 개발자의 끝이 아니라, 일하는 방식이 바뀐다는 의미)"

**회의론 (대체 안됨):**
- 많은 소프트웨어 엔지니어
- 기술적 이유 + 감정적 이유
- 대체 인정하기 싫음

### McKinsey 연구 결과

**AI가 개발자를 도울 수 있는 정도:**

```
문서화:          2배 생산성 ⭐⭐⭐⭐⭐
코드 생성:       25-50% 향상 ⭐⭐⭐⭐
코드 리팩토링:   25-50% 향상 ⭐⭐⭐⭐
복잡한 태스크:   최소한의 개선 ⭐
```

**추가 발견:**
- Frontend > Backend (AI가 더 잘함)
- 단순 작업 > 복잡한 작업

### 임팩트

**생산성:**
```
문서화: 2배
코드 생성: 25-50%
전체: 55% 증가 보고
일부 작업: 50% 시간 단축
```

**비즈니스 영향:**
```
더 적은 엔지니어로 더 많은 것 달성
→ 아웃소싱 산업 영향 (단순 작업)
```

**역할 변화:**
```
이전: 코드 작성자 (Writer)
현재: 코드 설계자 + 검토자 (Designer + Reviewer)

AI가 코드 생성 → 개발자가 검토/수정
```

## 2. Image and Video Production

> **"확률적 특성 덕분에 창작 태스크에 강함"**

### 성공적인 스타트업

**Midjourney:**
- 이미지 생성
- 1.5년 만에 $200M ARR (2023년 말)

**기타:**
- Adobe Firefly: 사진 편집
- Runway: 비디오 생성
- Pika Labs: 비디오 생성
- Sora: 비디오 생성

**Apple App Store (2023년 12월):**
```
Graphics & Design Top 10 무료 앱:
50%가 이름에 "AI" 포함

곧: AI가 기본 → 이름에서 "AI" 제거
```

### Text-to-Image

**1. DALL-E 3 (OpenAI)**

특징:
- GPT-4와 통합
- 프롬프트 자동 개선
- 높은 품질
- ChatGPT Plus에서 사용 가능

예시:
```
Prompt: "A photo of an astronaut riding a horse on Mars, 
         cinematic lighting, photorealistic"
→ [고품질 이미지 생성]
```

**2. Midjourney**

특징:
- 예술적 품질
- Discord 기반
- 커뮤니티 중심

강점:
```
✓ 아티스틱한 결과물
✓ 일관된 스타일
✓ 세밀한 디테일

약점:
✗ 프로그래밍 방식 접근 어려움
✗ API 없음 (현재)
```

예시:
```
/imagine prompt: fantasy castle on a floating island, 
sunset, Studio Ghibli style --ar 16:9 --v 6
```

**3. Stable Diffusion**

특징:
- 오픈소스
- 로컬 실행 가능
- 커스터마이징 가능

장점:
```
✅ 완전 무료
✅ 프라이버시 (로컬 실행)
✅ Finetuning 가능
✅ API 통합 용이
```

Finetuning 사례:
```
1. 특정 스타일 학습
   - 회사 브랜드 이미지
   - 특정 아티스트 스타일

2. 도메인 특화
   - 의료 이미지
   - 건축 디자인
   - 패션 디자인
```

### Text-to-Video

**1. Runway Gen-2**

능력:
```
Text → Video:
"A person walking through a neon-lit street"
→ 4초 비디오

Image → Video:
[이미지 업로드] + "Make the clouds move"
→ 움직이는 비디오
```

사용 사례:
- 영화 프리비주얼라이제이션
- 광고 스토리보드
- 소셜 미디어 콘텐츠

**2. Pika Labs**

특징:
- 텍스트 → 비디오
- 이미지 → 비디오
- 비디오 편집

**3. Sora (OpenAI, 2024)**

혁신적 능력:
```
- 최대 60초 비디오
- 1080p 고품질
- 물리적 일관성
- 복잡한 장면
```

예시:
```
Prompt: "A movie trailer featuring the adventures of 
the 30-year-old space man wearing a red wool knitted 
motorcycle helmet, blue sky, salt desert, cinematic 
style, shot on 35mm film, vivid colors"

→ 영화 트레일러 수준의 60초 비디오
```

### 소비자 사용 사례

**프로필 사진:**
- LinkedIn, TikTok용 AI 생성
- 취업 지원 시 첫인상 개선
- 인식 변화:
  - 2019 Facebook: AI 프로필 금지
  - 2023: 많은 앱이 AI 프로필 도구 제공

### 기업 사용 사례

**광고 & 마케팅:**

기업 예산의 11% = 마케팅 → 자동화 시 큰 절감

활용:
```
1. 직접 생성:
   - 프로모션 이미지/비디오

2. 브레인스토밍:
   - 아이디어 생성
   - 초안 작성 → 전문가 다듬기

3. A/B 테스트:
   - 여러 버전 생성
   - 성과 측정

4. 변형 생성:
   - 계절별 (가을→낙엽, 겨울→눈)
   - 지역별
```

### 창작 민주화

```
이전:
아이디어 → 디자이너 고용 → 수일~수주 → $수천~수만

현재:
아이디어 → AI 프롬프트 → 수초~수분 → $0~$수십

민주화: 전문가만 → 누구나
```

### 산업별 영향

**마케팅/광고:**
```
✅ 빠른 A/B 테스트
✅ 낮은 제작 비용
✅ 다양한 바리에이션
⚠️ 저작권 복잡성
```

**영화/엔터테인먼트:**
```
✅ 프리비주얼라이제이션
✅ VFX 초안
✅ 컨셉 아트
⚠️ 일자리 영향
```

**게임 개발:**
```
✅ 에셋 생성
✅ 텍스처 제작
✅ 컨셉 디자인
⚠️ 일관성 유지 어려움
```

## 3. Writing

> **"LLM은 text completion을 위해 학습되므로 writing을 잘함"**

### MIT 연구 (Noy and Zhang, 2023)

**실험 설계:**
- 453명 대학 졸업 전문가
- 직업별 writing 태스크
- 절반에게만 ChatGPT 노출

**결과:**
```
ChatGPT 사용 시:
✓ 작성 시간: 40% 감소
✓ 품질: 18% 향상
✓ 능력 격차 감소 (writing 약한 사람에게 더 도움)

실험 후:
- 2주 후 실제 업무 사용: 2배 증가
- 2개월 후: 1.6배 유지
```

### 소비자 사용 사례

**1. 커뮤니케이션 개선**
```
화난 이메일 → "pleasant하게 바꿔줘"
불릿 포인트 → 완전한 문단
```

많은 사람들:
> "중요한 이메일은 AI로 개선하지 않고 보내지 않는다"

**2. 학생**
- 에세이 작성
- ChatGPT 다운 → Discord에 불평

**3. 작가**
- AI로 책 작성
- 인터랙티브 책 (독자 선호에 따라 플롯 변경)
- 어린이 책: 어려운 단어 중심 스토리 생성

**4. 앱 통합**
- Google Docs
- Notion
- Gmail
- Grammarly: 유창함, 일관성, 명확성 개선

### 기업 사용 사례

**1. 영업/마케팅/커뮤니케이션**
```
- Cold outreach emails
- 광고 카피
- 제품 설명
- 성과 리포트 (많은 매니저 사용)
```

**2. CRM 도구**
- HubSpot
- Salesforce
- 웹 콘텐츠 생성
- Outreach 이메일

### SEO의 양날의 검

**AI가 SEO를 너무 잘함:**

이유:
```
AI 모델 학습 데이터 = 인터넷
인터넷 = SEO 최적화된 텍스트로 가득
→ AI가 SEO 패턴 학습
```

**Content Farm 문제:**

2023년 6월 NewsGuard 조사:
```
- 거의 400개 광고
- 141개 유명 브랜드
- AI 생성 쓰레기 웹사이트에 노출

극단적 사례:
- 한 웹사이트: 하루 1,200개 기사 생성

**Amazon 문제 (2023 NYT 보도):**
```
저품질 AI 생성 여행 가이드북 범람:

모두 AI 생성!

Chip Huyen의 가설:
> "인터넷 콘텐츠를 너무 불신하게 되어  
> 신뢰하는 사람/브랜드의 콘텐츠만 읽게 될 것"

### 주요 도구

**Jasper:**
```
용도: 마케팅 콘텐츠
- 블로그 포스트
- 광고 카피

특징:

**Copy.ai:**
```
용도: 카피라이팅
- 소셜 미디어

특징:
- 여러 바리에이션

```

**Grammarly:**
```
이전: 문법 체크

기능:
- 문법/스타일
- 톤 제안
- 명확성 개선
- 문장 재작성
```

### 작가의 역할 변화

```
전통적 작문:
1. Brainstorming (오래 걸림)
2. 아웃라인
3. 초안 (가장 오래)
4. 편집
5. 교정

AI 보조 작문:
1. Brainstorming (AI와 함께, 빠름)
2. 아웃라인 (AI 제안)
3. 초안 (AI 생성 → 검토/수정)
4. 편집 (AI 제안)
5. 교정 (AI 자동)

작가의 역할: 실행자 → 편집자 + 큐레이터
```

### 도전 과제

**1. Hallucination**
```
AI가 사실이 아닌 내용 생성
→ 팩트 체크 필수
```

**2. 표절**
```
AI 생성 콘텐츠 판별 어려움
학교/저널에서 문제
```

**3. 저작권**
```
AI 생성 콘텐츠의 저작권은?
상업적 사용 가능?
```

**4. 독창성**
```
AI 글은 "영혼"이 없다?
"AI 스타일"로 획일화?
독자가 구별 가능?
```

## 4. Education

### 학생들의 현실

```
ChatGPT 다운될 때:
→ OpenAI Discord 서버
→ 학생들 불평: "숙제 못 해요!" 😭
```

**초기 반응:**
- NYC 교육청: ChatGPT 금지 (치팅 우려)
- LA 교육청: ChatGPT 금지
- 몇 달 후: 결정 번복 ✓

### AI 활용 방법

**1. 개인화 학습**

기존 문제:
> "광고는 개인화되는데 (모두 다르다는 것을 아니까)  
> 교육은 왜 개인화 안 되나?"

AI 해결:
```
- 교재 요약
- 개인화된 강의 계획
- 포맷 맞춤:
  ├─ 청각 학습자 → 읽어주기
  ├─ 동물 좋아하는 학생 → 동물 중심 예시
  └─ 코드 선호 → 수식을 코드로
```

**2. 언어 학습**

특히 효과적:
- AI와 롤플레이
- 다양한 시나리오 연습
- 실시간 피드백

**Duolingo 연구 (Pajak & Bicknell, 2022):**
```
코스 제작 4단계 중:
"Lesson personalization"이 AI로 가장 큰 혜택

AI가 각 학생에 맞게 문제 조정
```

### Khan Academy + GPT-4: Khanmigo

**기능:**

**1. 개인화된 튜터**
```
학생: "방정식을 못 풀겠어요"
AI: "어디서 막혔나요? 한 단계씩 같이 해봅시다"
```

**2. Socratic 대화**

답을 직접 주지 않고:
```
✗ 나쁜 예:

✓ 좋은 예 (Khanmigo):

AI: "좋은 질문이에요! 먼저 x만 남기려면?"

```

Duolingo Max:

  ├─ 보통 → 나중에 요약
  └─ + 관련 문서 자동 첨부
```

### 실제 사용 사례

**1. 고객 지원 자동화**

워크플로우:
```
고객 이메일
    ↓
AI 내용 분석
    ↓
├─ 간단 → AI 자동 답장
├─ 복잡 → 티켓 + 담당자
└─ 긴급 → 즉시 알림 + 에스컬레이션

결과:
- 응답 시간 80% ↓
- 만족도 ↑
- 직원은 복잡한 문제에 집중
```

**2. 콘텐츠 제작 파이프라인**

워크플로우:
```
1. 뉴스/트렌드 모니터링 (자동)
2. AI가 관련성 판단
3. 관련 → 초안 생성
4. 사람 검토/수정
5. 자동 포맷팅
6. 소셜 미디어 예약

결과:
- 생산량 3배
- 품질 유지
- 시간 70% 절약
```

**3. 데이터 수집 & 리포팅**

워크플로우:
```
1. 여러 소스 데이터 수집 (API)
2. AI 정규화
3. 인사이트 추출 (AI)
4. 차트 자동 생성
5. 리포트 초안 (AI)
6. 이메일/Slack 전송

주기: 매일/매주 자동

결과:
- 수동 작업 제로
- 최신 데이터
- 시간 90% 절약
```

**4. 영업 워크플로우**

워크플로우:
```
1. 새 리드 (CRM)
2. AI 프로파일 분석
   - 회사 크기
   - 산업
   - 예산
3. 적격 여부 판단
4. 적격:
   - 개인화 이메일 (AI)
   - 담당자 배정
   - 일정 생성
5. 부적격:
   - Nurture 캠페인

결과:
- 응답 시간 단축
- 전환율 ↑
- Qualified 리드에만 집중
```

### AI Agent의 등장

**차이:**
```
전통적 자동화:
- 미리 정의된 규칙
- 순차적 실행
- 정적

AI Agent:
- 목표 기반
- 동적 의사결정
- 자율적
```

**예시: AutoGPT 개념**
```
목표: "경쟁사 분석 리포트"

AI Agent 자율 실행:
1. "경쟁사 리스트 필요" → 웹 검색
2. "각 웹사이트 분석" → 스크래핑
3. "뉴스 검색" → API 호출
4. "데이터 정리" → 코드 실행
5. "리포트 작성" → 문서 생성

→ 사람처럼 스스로 계획하고 실행
```

### 임팩트

**생산성:**
```
반복 작업 자동화:
- 데이터 입력
- 파일 정리
- 리포트 생성
- 이메일 응답

→ 창의적 작업에 집중
```

**비용 절감:**
```
예:
월 40시간 반복 작업 × $50/시간
= $2,000/월

자동화 후:
월 5시간 (모니터링) = $250/월

절감: $1,750/월 ($21K/년)
```

**에러 감소:**
```
인간: 피곤 → 실수
AI: 일관된 품질
```

**24/7 운영:**
```
인간: 8시간/일
AI: 24시간/일

→ 빠른 응답
→ 글로벌 운영
```

### 도전 과제

**1. 과도한 자동화**
```
모든 것 자동화:
- 인간 터치 상실
- 에러 시 큰 영향
- 통제력 상실

균형 필요
```

**2. 유지보수**
```
자동화 많아지면:
- 관리 복잡도 ↑
- 버그 찾기 어려움
- 문서화 필요
```

**3. 변화 관리**
```
직원: "내 일자리가..."

필요:
- 교육
- 역할 재정의
- 새 기회 제시
```

## 핵심 요약

### 8가지 Use Case 공통 패턴

**1. 민주화 (Democratization)**
```
전문가만 → 누구나

예:
- 코딩: 프로그래머 → 누구나 (AI 도움)
- 디자인: 디자이너 → 누구나 (AI 생성)
- 작문: 작가 → 누구나 (AI 보조)
```

**2. 역할 변화 (Role Shift)**
```
실행자 (Doer) → 검토자 + 큐레이터 (Reviewer + Curator)

AI가 초안/옵션 생성
→ 인간이 선택/수정/개선
```

**3. 속도 (Speed)**
```
대부분 작업: 80-90% 시간 단축

예:
- 코딩: 25-100%
- 작문: 40%
- 정보 찾기: 85-90%
```

**4. 접근성 (Accessibility)**
```
- API로 쉽게 접근
- 낮은 비용
- 낮은 기술 장벽

→ 작은 팀/개인도 강력한 AI 활용
```

| Data Organization | 주 5-10h | Notion AI, M365 | - |
| Workflow | 80% 응답 시간↓ | Zapier AI | - |

### 직업 노출도 (Eloundou et al., 2023)

**0% 노출:**
